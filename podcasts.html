Linear Search vs Binary Search

May 26th 2022 4 Min read

Searching Algorithms are used to find whether a specific group of data exists within a data structure. If it does exist, a search algorithm will find it and retrieve the information. There are many diverse types of search algorithms. We will look at two well-known algorithms for searching: Linear and Binary search algorithms.

Linear Search is a search algorithm that begins at one end of a data structure, checking each data item in turn until the required item is found, or the end of the structure is reached. If data is unordered and irregular, the most sensible way to solve this problem is linear search algorithm. Linear searches start at the beginning and sequentially look at each item in a list until you find the one you are looking for or until all of the items have been searched. If the data is at the beginning of the list, the search process is completed in a short time, but if it is towards the end of the list and the data set is large, the search process may take a long time. Linear Searches are relatively simple, but they are inefficient when searching through long lists.

Binary Search algorithm that begins in the middle of a data structure, cutting half of the remaining data with each pass. Binary searches are only appropriate when the data is sorted. If the data is sequential, the most efficient search method is the binary search algorithm. It works by repeatedly dividing in half the part of the data list that could contain the required data item. The binary search algorithm is quite efficient because half of the data that is not searched is eliminated in one go. If a list of one million elements were to be searched using a binary search, it would take no more than 20 iterations to find a particular piece of data or discover that the data is not in the list.

The linear search algorithm is fine for small lists, but very inefficient for large lists. In contrast, the binary search algorithm is extremely efficient. Each time an item is examined, if it is not the correct item, half of the list is discarded. At this point, it is necessary to look at the list and choose the most suitable algorithm according to it.
